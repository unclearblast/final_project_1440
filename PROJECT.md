# OrbitaMarket – План проекта

## Цель
Разработать ядро платформы OrbitaMarket для продажи спутниковых продуктов: приём заказов (архивные снимки, будущая съёмка, подписки), биллинг в геокредитах, асинхронная обработка оплаты с гарантиями идемпотентности и конкурентной корректности баланса.

## Стейкхолдеры
- **Оператор ДЗЗ** – заказывает будущую съёмку, архивные снимки и мониторинг.
- **Аналитическая компания** – покупает готовые продукты мониторинга.
- **Администратор платформы** – управляет тарифами, пользователями и отслеживает статистику.

## Текущий статус
- Все микросервисы (Payments, Orders, Gateway) реализованы и проходят ручное тестирование.
- Автотесты покрывают 14 сценариев через Gateway (отчёт Allure).
- SQL-запрос для аналитики «кто сколько купил» готов.
- C4-диаграммы (System Context, Containers) созданы.
- Инструменты ИБ (Gitleaks, Semgrep, OSV) запущены, таблица триажа содержит более 20 находок с классификацией TP/FP/риск.
- Асинхронное взаимодействие через Kafka частично отлажено, требуется финальная проверка сквозного сценария.

## Риски и проблемы
- Нестабильность сети в среде разработки ограничивала возможности Semgrep (использовались локальные правила) и OSV (часть зависимостей не обогащена).
- Для полной демонстрации асинхронной оплаты необходимо поднять Docker-окружение (Kafka + БД) и все три сервиса.

## Используемые технологии

- PostgreSQL 15
- Apache Kafka (Confluent 7.5.0)
- Docker, Docker Compose
- Maven, JUnit 5, RestAssured, Allure
- Gitleaks, Semgrep, OSV-Scanner
