# Чек-лист сценариев тестирования OrbitaMarket

| № | Сценарий | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Happy path: создать счёт, пополнить на 1000, заказ на 120 | Статус заказа PAID, баланс 880 |
| 2 | Недостаточно средств: баланс 50, заказ на 120 | Статус заказа PAYMENT_FAILED, баланс 50 |
| 3 | Повторный OrderPaymentRequested с тем же order_id | Баланс не списан повторно (идемпотентность) |
| 4 | Два заказа по 400 при балансе 1000 | Итог баланса 200, статусы PAID, нет отрицательного баланса |
| 5 | Повторный POST /accounts для того же user_id | Дубликат счёта не создаётся (200 или 409) |
| 6 | Создание заказа без обязательных полей | 400 INVALID_PAYLOAD |
| 7 | Создание заказа с price ≤ 0 | 400 INVALID_PRICE |
| 8 | Создание заказа с неизвестным product_type | 400 UNKNOWN_PRODUCT_TYPE |
| 9 | Получение несуществующего счёта | 404 ACCOUNT_NOT_FOUND |
| 10 | Пополнение несуществующего счёта | 404 ACCOUNT_NOT_FOUND |
| 11 | Пополнение с amount ≤ 0 | 400 INVALID_AMOUNT |
| 12 | Запрос без заголовка X-User-Id | 400 MISSING_USER_ID |
| 13 | Получение списка заказов | Возвращается список заказов пользователя |
| 14 | Получение конкретного заказа | Возвращается детали заказа |
| 15 | Получение чужого заказа | 403 или 404 |
